Big O Notation

Here is a cheat-sheet of the Big O notation

  • O(1) Constant – no loops
  • O(log N) Logarithmic – usually searching algorithms have log n if they are sorted (Binary Search
  • O(n) Linear – for loops, while loops through n items

The Four Rules of Big O Notation

  • Worst Case:

    - Big O only cares about worst case

  • Remove Constants:

    - Constants are not important in Big O notation because the concern is scalability.

    For example, if you have a function that has a running time of 5n, we say that this function runs on the order of the big O of N. This is because as N gets large, the 5 no longer matters

  • Different Terms for inputs:

    the same terms do not apply if the inputs are different.

    So the terms will depend on the different loops inputs array size

  • Drop Non dominant:
